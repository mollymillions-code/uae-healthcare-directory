# Zavis Landing - Project Architecture

## What This Is
Zavis Landing is the main website for zavis.ai, a healthcare directory and research ecosystem. It serves provider listings, city/facet pages, search, and research content.

## Tech Stack
- **Frontend/Backend:** Next.js 14.2.35 (App Router), React 18, TypeScript, Tailwind CSS
- **Database:** PostgreSQL 16 (local on EC2) via node-postgres (`pg`)
- **ORM:** Drizzle ORM (`drizzle.config.ts` at root, migrations in `data/`)
- **Hosting:** AWS EC2 (13.205.197.148), PM2 process `zavis-landing` on port 3200, Nginx reverse proxy

## Directory Layout
- `src/` - Application source (Next.js App Router)
- `data/` - Data files and DB-related assets
- `scripts/` - Utility and migration scripts
- `public/` - Static assets
- `remotion/` - Video generation (Remotion)
- `drizzle.config.ts` - Drizzle ORM configuration

## Deployment
Push to `main` triggers a GitHub webhook (port 9100) that runs git pull, builds, and restarts the PM2 process. No CI/CD pipeline beyond the webhook.
