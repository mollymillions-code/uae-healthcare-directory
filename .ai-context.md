# Zavis Landing - Project Architecture

> **⛔ THIS IS A PRODUCTION SITE (zavis.ai). Every push to `main` deploys directly. Read `CLAUDE.md` before writing any code.**

## What This Is
Zavis Landing is the main website for zavis.ai — a healthcare directory and research ecosystem for the UAE. It serves provider listings, city/facet pages, search, industry intelligence articles, and a research content pipeline with social media distribution.

## Mandatory Pre-Push Checklist
1. `npm run lint` — must pass with zero errors
2. `npm run build` — must succeed if you changed/added pages
3. No unused imports — this crashed production on 2026-03-25
4. No `@neondatabase/serverless` — use `pg` (node-postgres) only
5. Full rules: `CLAUDE.md`, `.ai-collab/STATUS.md`

## Tech Stack
- **Frontend/Backend:** Next.js 14.2.35 (App Router), React 18, TypeScript, Tailwind CSS
- **Database:** PostgreSQL 16 (local on EC2) via node-postgres (`pg`) + Drizzle ORM
- **Hosting:** AWS EC2 (13.205.197.148), PM2 process `zavis-landing` on port 3200, Nginx reverse proxy
- **Storage:** Cloudflare R2 for images (`zavis-images` bucket)
- **AI/Content:** Gemini API (content generation), Anthropic Claude (research), ElevenLabs (voiceover)
- **Social:** Postiz API (LinkedIn scheduling), Resend (email/newsletter)

## Directory Layout
- `src/app/(landing)/` — Main marketing pages (home, about, pricing, etc.)
- `src/app/(directory)/` — Healthcare directory (cities, categories, providers, search, intelligence, labs, dashboard)
- `src/app/(research)/` — Research ecosystem dashboard
- `src/app/api/` — API routes (search, research pipeline, automation, emails, posts, intelligence, health)
- `src/lib/db/` — Drizzle ORM setup, schema, seed scripts — uses `pg`, NOT neon
- `src/lib/research/` — Research platform (DB, auth, postiz, email, schema.sql) — uses `pg`, NOT neon
- `src/lib/intelligence/` — Journal article automation (pipeline, data, categories, SEO)
- `scripts/` — CLI scripts — ALL use `pg`, NOT neon
- `scripts/automation/` — Cron orchestrator, daily/weekly distribution, health checks
- `remotion/` — Video generation framework
- `public/` — Static assets, PDFs, reports

## Database
- **Connection:** `import { db } from "@/lib/db"` for Drizzle ORM, `import { Pool } from "pg"` for raw SQL
- **Driver:** `pg` (node-postgres) — NOT `@neondatabase/serverless`
- Two schema layers:
  1. **Drizzle ORM** (`src/lib/db/schema.ts`): cities, areas, categories, subcategories, providers, faqs, journal_articles, journal_subscribers, google_reviews, claim_requests, provider_categories
  2. **Raw SQL** (`src/lib/research/schema.sql`): pipeline_runs, pipeline_comments, linkedin_posts, email_blasts, performance_scores, automation_schedules, automation_runs, post_queue, automation_notifications, performance_insights

## Deployment
- **CI pipeline:** Lint + type-check → SSH deploy to EC2 → build → health check
- **Safe deploy:** Build failure auto-rolls back to last good build (site stays up)
- **Health check:** `/api/health` returns git SHA — auto-rollback if site doesn't respond
- **All scripts/workflows run ON EC2** via SSH (DB is localhost-only)

## Critical Nginx Note
`_next/static` files MUST be served directly from disk by Nginx (not proxied to Next.js). Route groups like `(directory)` create chunk paths with literal parentheses that browsers URL-encode as `%28/%29`, which Next.js cannot resolve.
