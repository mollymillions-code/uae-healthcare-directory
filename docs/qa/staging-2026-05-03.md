## Findings (after my mid-session fixes)

### BLOCKERs

#### B1 — `/jobs/login`, `/jobs/signup`, `/jobs/profile` rendered as 404 content on first staged build
**Symptom:** HTTP 200 returned, but the SSR'd HTML body contained `next-error` markers and the text "404: This page could not be found." The candidate funnel was effectively dead — every "Create profile" CTA led to a non-functional page.

**Root cause hypothesis:** Some interaction between the Suspense-wrapped `useSearchParams()` client components and Next.js 14's static prerender produced 404 content. The fix in commit `f63f666a` adds explicit per-route `layout.tsx` files with `robots: { index: false, follow: false }` metadata, removes login/signup from the sitemap, and fixes title-template double-suffixing — those changes are now staging.

**Re-test required:** After the new stage finishes, the `/jobs/login`, `/jobs/signup`, `/jobs/profile` URLs need verification. If still broken, root cause is something else and needs deeper investigation.
