## 4. Inconsistent flows

### 4.1 Admin uses client-side sessionStorage auth — MEDIUM
**File:** `src/app/(directory)/admin/AdminShell.tsx:26-68`
**Issue:** The admin dashboard validates the dashboard key client-side and stores in sessionStorage. This means:
- Auth state lost on tab close
- Sensitive admin operations gated only by client-side key validation
- API endpoints have their own server-side gate via `validateAdminAuth` middleware (see `src/lib/admin-auth.ts`), so the backend is fine
- The UX is the only fragile part: if user loses sessionStorage, they have to re-enter the key

**Fix:** Defer. Migrating to server-side cookie-based auth is a separate project. Not exploitable.

### 4.2 Consumer + provider portal redirects — both working
- `/account/*` → `redirect("/login?redirect=/account")` ✓
- `/provider-portal` → `redirect("/provider-portal/login?redirect=/provider-portal")` ✓
- `/dashboard/*` → password-gated via `/dashboard-auth` ✓

## 6. Failing backreferences

### 6.1 Terms / privacy URL inconsistency — MEDIUM
**Issue:** Codebase uses two URL conventions for the same pages:
- Some files link to `/terms-of-service` + `/privacy-policy`
- Others to `/terms` + `/privacy`

All four routes exist and serve content (verified). But the inconsistency means SEO sees duplicate content unless one canonicalizes to the other.

**Files with `/terms-of-service` + `/privacy-policy`:**
- `src/components/landing/layout/Footer.tsx:131-135`

**Files with `/terms` + `/privacy`:**
- `src/components/layout/Footer.tsx:100-101`
- `src/components/landing/book-a-demo/DemoForm.tsx`

**Fix:** Add `next.config.mjs` redirects from `/terms` → `/terms-of-service` and `/privacy` → `/privacy-policy`. Standardize all internal links on the longer form.
